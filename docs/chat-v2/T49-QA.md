# T49 — QA (роль і стать у списку «Люди»)

## Реалізація

- **Ролі:** іконки з `chat_role` у presence / `auth/user` — щит червоний (адмін), зелений (модератор), помаранчевий плюс у колі (VIP); звичайний користувач і гість без іконки ролі. Компонент `ChatPeerRoleIcon.vue`, кольори через токени `--rp-peer-icon-*` у `app.css`.
- **Стать:** символи ♂ / ♀ / ⚧ з підписами у `title` та `aria-label` (українською). Для **інших** учасників — лише після `GET /api/v1/rooms/{room}/peer-hints?user_ids=…` (зареєстрований переглядач; прихований профіль / `prefer_not` / гість-ціль не потрапляють у відповідь). **Гість-переглядач** отримує порожній `data` — на клієнті не показуємо стать пірів (політика T22/T49).
- **Власний рядок:** стать з `user.profile.sex`, якщо не `sex_hidden`.

## Автоматичні перевірки

```bash
cd backend && php artisan test --filter=RoomPeerHintsTest
cd backend && php artisan test
cd backend && npm run build
```

## Ручний сценарій (опційно)

1. Зареєстрований користувач з заповненою неприхованою статтю в профілі — у списку «Люди» біля ніку з’являється символ.
2. Акаунти admin / moderator / vip — відповідні іконки.
3. Увійти як гість — символів статі для інших немає (навіть якщо вони зареєстровані).
