# T34 — QA: вставка зображення з буфера в композер

## Реалізація

- `paste` на `#chat-composer`: якщо в `clipboardData` є **file** з дозволеним MIME (`image/jpeg`, `image/png`, `image/gif`, `image/webp`), подія **скасовується** і виконується той самий `POST /api/v1/images`, що й для file picker (`uploadChatImageFile`).
- Спільна валідація розміру (макс. 4 МБ) і MIME у `utils/chatComposerImageUpload.js`; помилки показуються в тому ж блоці, що й для вибору файлу.
- **Гість:** зображення з буфера не завантажується; показується пояснення (без зміни бекенду).
- **Лише текст** у буфері або незображення — `preventDefault` **не** викликається, звичайний paste тексту в textarea.
- Placeholder композера оновлено короткою підказкою про paste.

## Ручний чекліст

- Зареєстрований користувач, відкрита кімната: скріншот / копія картинки → **Ctrl+V** (або **⌘V**) у полі → прев’ю вкладення, можна надіслати разом із текстом.
- Вставка звичайного тексту без змін.
- Вставка зображення під час активного завантаження або надсилання → повідомлення про очікування.

## Автоматичні перевірки

- Логіка вибору файлу з буфера винесена в `getFirstClipboardImageFile` (зручно для майбутнього JS-тесту); у репо немає Vitest/Jest — автотест paste не додано.
- `npm run build` — PASS
- `php artisan test` — 100 passed (API зображень без змін)
