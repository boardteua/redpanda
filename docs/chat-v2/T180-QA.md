# T180 — QA: маршрутизація моделей «Руда Панда» (Flash / Pro / Flash-Lite)

**Дата:** 2026-04-01  
**Вердикт:** PASS

## Що зроблено

- Евристичний **`RudaPandaIntentClassifier`**: `simple` / `complex` / `image` (пріоритет: image → complex → simple).
- **`RudaPandaModelRouter`**: вибір `modelId` за наміром і роллю (гість / зареєстрований / VIP), узгоджено з **T176** (Pro лише для VIP + complex).
- **`RudaPandaModelRoute`**: опційний merge `generationConfig` ( **`thinkingConfig`** для Pro: `GEMINI_PRO_THINKING_LEVEL` або `GEMINI_PRO_THINKING_BUDGET`, див. [thinking mode](https://ai.google.dev/gemini-api/docs/thinking-mode)).
- Конфіг **`config/services.php` → gemini**: `model_flash_lite`, `model_flash`, `model_pro`, `model_image`, knobs для thinking; змінні в **`.env.example`**.

cd backend && php artisan test --filter RudaPanda

## Дефолти оператора

| Змінна | Поведінка якщо порожньо |
|--------|-------------------------|
| `GEMINI_MODEL_FLASH` | береться `GEMINI_DEFAULT_MODEL` |
| `GEMINI_MODEL_FLASH_LITE` | fallback на значення flash-tier |
| `GEMINI_MODEL_PRO` | `gemini-2.5-pro` |
| `GEMINI_MODEL_IMAGE` | fallback на flash-tier (уточнення в **T181**) |

## Нотатка інтеграції

Пайплайн, що викликає `GeminiClient::generateContent`, має передавати **`$route->modelId`** і застосовувати **`$route->mergeIntoPayload($payload)`** перед HTTP-викликом. Окремий job для повної відповіді бота може з’явитись у наступних тасках епіку; T180 постачає **контракт маршрутизації** та лог `ruda-panda model routed` у каналі **structured** (метадані без тексту користувача).
