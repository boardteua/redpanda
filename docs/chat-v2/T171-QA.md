# T171 — QA: гучність сповіщень після збереження в профілі

**Вердикт:** PASS (логіка оновлення `user` після збереження профілю)

## RCA (коротко)

`UserProfileModal` емить `updated` з повним `data.data` (`UserResource`), але `ChatRoomModals` ретранслював `profile-updated` **без** цього аргументу. У `ChatRoom.onProfileModalUpdated(nextUser)` `nextUser` був `undefined`, тож `this.user.notification_sound_prefs` не оновлювався після збереження звуків — відтворення лишалося з початковим `volume_percent`.

## Фікс

- `ChatRoomModals.vue`: `@updated="$emit('profile-updated', $event)"`.

## Перевірка (ручна)

1. Увійти в чат, відкрити профіль → вкладка звуків.
2. Встановити гучність ~10%, зберегти; іншим акаунтом надіслати повідомлення в кімнату / приват / згадку — звук **тихий**.
3. Підняти гучність ~100%, зберегти — наступні сповіщення **гучніші** (без F5).
4. Перезавантажити сторінку — поведінка збігається зі збереженим значенням.

## Автоматизовані команди

- `cd backend && npm run build` — PASS
