# T54 — QA: керування кімнатами (модал + CRUD API)

## Статус

**QA PASS** — автоматизовані перевірки нижче виконані в CI/локально.

## Що змінилось

- **API:** `PATCH /api/v1/rooms/{room}`, `DELETE /api/v1/rooms/{room}`; у відповідях кімнати — `created_by_user_id`, `messages_count` (у списку та після створення/оновлення).
- **Політика:** назва/опис — творець або модератор/адмін; `access` — лише модератор/адмін; видалення — творець або staff; **DELETE** заборонено, якщо є повідомлення (**422**, `room_has_messages`).
- **UI:** кнопка «Додати кімнату» відкриває модал лише з формою створення; у списку кімнат для творця/staff — олівець → окремий модал редагування/видалення (підтвердження видалення — `ConfirmDialogModal`, z-index 95).

## Докази

```bash
cd backend && php artisan test --filter=Room
```

Очікування: усі тести з фільтром `Room` — **PASS** (у т.ч. `RoomCrudApiTest`, `RoomCreateApiTest`, решта кімнатних suite).

```bash
cd backend && npm run build
```

Очікування: збірка Vite — **PASS**.

## A11y (короткий чекліст)

- Відкриття модалу з кнопки на вкладці «Кімнати»; **Escape** закриває; **Tab** циклить усередині діалогу (`RpModal`).
- Після закриття модалу фокус повертається до тригера (поведінка `RpModal`).

## Примітки

- Поле `created_by_user_id` заповнюється при `POST /rooms`; legacy-рядки залишаються з `null` — їх текст редагує лише staff.
