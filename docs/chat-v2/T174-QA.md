# T174 — Load older history on scroll (room + private) — QA

**Статус:** PASS (2026-03-31)  
**Зміна:** Підвантаження старіших повідомлень “по скролу вгору” для кімнати та привату з компенсацією `scrollTop` (без стрибка). Розмір чанка \(N\) — user pref у профілі. Повідомлення в сегменті після `New messages` мають окрему візуальну підсвітку.

## Автоматичні перевірки

| Крок | Команда | Результат |
|------|---------|-----------|
| Backend tests (scope) | `cd backend && php artisan test tests/Feature/ChatApiTest.php tests/Feature/PrivateMessageApiTest.php` | PASS |
| Front build | `cd backend && npm run build` | PASS |

## Ручний сценарій

### Кімната

1. Відкрити кімнату з історією (щоб було що підвантажувати).
2. Прокрутити стрічку **до абсолютного верху**.
3. PASS-критерії:
   - підвантажується рівно \(N\) старіших (або менше, якщо історія закінчилась),
   - **позиція не стрибає** (після prepend на екрані залишається той самий “якірний” меседж),
   - повторити кілька разів — в кінці історії UI більше не робить зайвих запитів (керується `meta.has_more_older`).

### Приват

1. Відкрити приват з користувачем, де є історія.
2. Прокрутити список **до абсолютного верху**.
3. PASS-критерії — аналогічно кімнаті: prepend без стрибка, коректний “no more”.
