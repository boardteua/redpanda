# T61 — QA evidence

**Задача:** ховати кнопку панелі при відкритому сайдбарі; хедер кімнати з акцентом «поточної» кімнати.

## Build

- `cd backend && npm run build` — **PASS** (2025-03-22, vite v7.3.1).

## Реалізація (коротко)

- **Кнопки панелі:** `v-if="!panelOpen"` у `ChatRoomHeader.vue` (mobile + desktop); при відкритій панелі з DOM прибрані, щоб не потрапляли в Tab.
- **Фокус:** у `beginOpeningPanel` та `selectRoom` після закриття панелі ціль фокусу — `#main-content`, якщо тригер був кнопкою панелі (після T61 ref кнопки зникає з DOM під час відкритого сайдбару).
- **Колір кімнати:** API не містить поля кольору кімнати — стабільний акцент за `room_id % 7` (палітра hex у `ChatRoom.vue`): `color-mix` тла + `inset` смуга 4px у `ChatRoomHeader.vue`; текст лишається на токенах `--rp-chat-sidebar-fg` / `--rp-chat-sidebar-muted` (WCAG AA на незмінних кольорах тексту).

## UI / a11y (оператор)

- [ ] Сайдбар закритий → кнопка панелі видима.
- [ ] Сайдбар відкритий (mobile + desktop) → кнопок панелі в хедері немає; закриття через X сайдбару / overlay / Escape (mobile) працює.
- [ ] Після закриття панелі Tab доходить до кнопки знову; фокус не «губиться» (очікувано — переведення на `#main-content` після сценарію відкриття з кнопки).
- [ ] Дві кімнати з різними `room_id` — різний відтінок акценту в хедері (світла / темна тема).
