# T194 — QA: `open_chat_settings` у URL

**Статус:** PASS (2026-04-01).

## Перевірки

1. **Deep link:** адмін відкриває `/chat/...?open_chat_settings=1` — модал налаштувань відкривається, у рядку браузера параметр `open_chat_settings` зникає одразу після bootstrap (разом з існуючим `consumeOpenChatSettingsQuery()`).
2. **Закриття:** після відкриття модалу з меню (або якщо параметр залишився в URL з будь-якої причини) — ESC / оверлей / скасування прибирають `open_chat_settings` з query (`stripOpenChatSettingsFromRouteIfPresent`).
3. **Збереження:** після успішного збереження налаштувань — той самий strip; **F5** не відкриває модал знову.

## Збірка

- `npm run build` (backend / Vite) — без помилок.
