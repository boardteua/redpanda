# T170 — PWA / Web Push: синхронізація стрічки після відкриття з push

## Симптом

У встановленій PWA (або з тим самим service worker) після кліку по push відкривається чат / потрібна кімната або приват, але **нове повідомлення не з’являється в стрічці**, хоча заголовок/текст у нотифікації коректні.

## RCA (корінь)

1. **Той самий URL / кімната.** Service worker (`resources/js/sw.js`) для існуючого клієнта викликає `client.navigate(targetUrl)` і `focus()`. Якщо користувач **вже** на цільовій кімнаті, Vue Router **не** змінює маршрут → не викликається `applyRoomSelection()` → `loadMessages()` знову не виконується.
2. **WebSocket у фоні.** У фоновій вкладці / згорнутій PWA подія `.MessagePosted` по Echo **може не потрапити** в клієнт (з’єднання призупинене або події втрачені). Після фокусу канал може знову працювати, але **пропущене** повідомлення вже не «доганяється» само.
3. **Poll лише при `wsDegraded`.** `pollNewMessages()` раніше виходив одразу, якщо `!wsDegraded`, тож періодичний REST-fallback **не** допомагав при «здоровому» WS, який просто пропустив події у фоні.

## Виправлення (мінімальне)

У `ChatRoom.vue`:

- Винесено **`fetchLatestRoomMessagesMerged()`** — той самий REST `GET .../messages?limit=80` + `mergeMessage` (ідемпотентно для `post_id`, узгоджено з T06 / T04).
- **`pollNewMessages()`** викликає цей метод лише коли `wsDegraded` (поведінка poll збережена).
- Після переходу **`document.visibilityState` → `visible`** (і при **`pageshow` з `persisted`** для bfcache) планується **debounced** (~400 ms) **`runFeedSyncAfterResume()`**:
  - відкритий приват → `loadPrivateMessages()` (якщо не в процесі завантаження);
  - інакше → `fetchLatestRoomMessagesMerged({ playSoundsForNew: false })`, щоб не дублювати звук після вже показаного push.

Перед синхронізацією викликається `ensureSanctum()`.

## Сценарії перевірки

- **Кімната:** відкрити потрібну кімнату, згорнути PWA / перевести вкладку у фон; з іншого акаунта надіслати повідомлення; клік по push → повідомлення **є** в стрічці без F5.
- **Приват:** аналогічно з `?private_peer=...` у URL push.
- **Той самий URL:** користувач уже в цій кімнаті, вкладка у фоні → push → focus → стрічка оновлюється.
