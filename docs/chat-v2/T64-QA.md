# T64 — QA: прямі посилання на MP4 → inline `<video>` (muted)

## Вердикт

**PASS** (автотести парсера + збірка; повна перевірка відтворення в браузері — за оператором, якщо потрібен живий MP4 URL).

## Доказ

- **Парсер (Node):** `node --test resources/js/utils/chatMessageBodyParse.test.mjs` — усі тести PASS; покрито `isLikelyMp4Url`, сегмент `inlineVideo`, `classifyUrl`, `messageHasBlockMedia`, відхилення небезпечних URL з credentials.
- **Збірка SPA:** `npm run build` (каталог `backend/`) — без помилок.

## Поведінка

- У стрічці / приваті: безпечний `https?://…` з шляхом, що закінчується на `.mp4` (регістр неважливий, дозволено query), рендериться як `<video muted controls playsinline preload="metadata">` у межах `max-w-lg` (як блокові ембеди).
- **Архів:** той самий URL показується як звичайне посилання (без плеєра).
- Звук за замовчуванням вимкнено (`muted`); користувач може вмикати через контроли браузера після взаємодії, якщо джерело не обмежує CORS/інше.

## Ручний сценарій (опційно)

Надіслати в кімнату повідомлення з публічним прямим URL на `.mp4`; переконатися, що відео з’являється без автозвуку, Tab до controls працює в межах підтримки браузера.
