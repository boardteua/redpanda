# Тернопільський Анонімний Чат — контекст для інтеграторів і LLM

Цей документ описує, як **без виконання JavaScript** у браузері зрозміти продукт, автентифікацію та основні HTTP-флоу. Канонічний контракт — [`/docs/openapi.yaml`](/docs/openapi.yaml). Розширений індекс вимог (для планування в репозиторії) — [`/docs/project-specs/chat-v2-setup.md`](/docs/project-specs/chat-v2-setup.md).

## Політика безпеки та приватності

- **Не** включайте до публічних відповідей **секрети** (файли конфігурації з обліковими даними, ключі API, паролі, токени сесій).
- **Не** публікуйте **PII** користувачів (email, IP, повні профілі), якщо це не ваш власний тестовий акаунт у ізольованому середовищі.
- **Staff / модерація:** ендпоінти з обмеженням за ролями **не** є публічним контекстом для загального LLM-індексу; описуйте їх лише на рівні «існують і вимагають відповідних прав», без URL-ів з реальними id.
- Публічні огляди для машин: **`GET /llms.txt`**, **`GET /docs/chat-v2/AI-AGENT-FRIENDLY.md`** (цей файл).

## Продукт (коротко)

- **Наступник** за сценаріями [board.te.ua](https://www.board.te.ua/): кімнатний чат, приватні повідомлення, друзі, архів, профіль.
- **Реалізація:** типовий вебзастосунок (серверний API + клієнтський UI у браузері). **Конкретні технології та версії** навмисно не фіксуються тут — вони змінюються й так частково визначаються з відповідей HTTP; для інтеграції достатньо **OpenAPI** та цього огляду.
- **Джерело правди для повідомлень:** HTTP API (ідемпотентний прийом з ідентифікатором повідомлення в контракті); канал real-time у браузері описаний у OpenAPI (`info.description` тощо), **без** прив’язки до внутрішніх назв сервісів у цьому тексті.

## Що доступно без JS

| Ресурс | Призначення |
|--------|-------------|
| `GET /llms.txt` | Markdown-огляд + посилання |
| `GET /docs/openapi.yaml` | OpenAPI 3 — основне джерело для агентів |
| `GET /docs/chat-v2/AI-AGENT-FRIENDLY.md` | Цей документ |
| `GET /docs/project-specs/chat-v2-setup.md` | Індекс специфікації (публічна копія) |
| `GET /api/v1/landing` | Публічні поля вітальні (без секретів) |
| `GET /up`, `GET /health/ready` | Liveness / readiness |
| `POST /api/v1/auth/*` | Реєстрація, логін, гість (див. OpenAPI) |

Канонічний URL чату після T153: **`/chat/{roomSlug}`**. Сумісність для legacy deep link: `?room=` (числовий id або slug) підтримується і канонізується клієнтом.

**Що вимагає JS у браузері:** повноцінний інтерактивний чат і оновлення в реальному часі в UI. Для агентів компенсуйте викликами **REST** за OpenAPI та полями `info.description`.

## Авторизація (cookie-сесія + гість)

1. Для клієнта з того ж origin: отримати CSRF-cookie за маршрутом і заголовками, описаними в **OpenAPI** (типово `GET /sanctum/csrf-cookie` з `credentials: include`).
2. Змінюючі запити: заголовок `X-XSRF-TOKEN` (значення з cookie `XSRF-TOKEN`, URL-decoded) + cookie сесії.
3. **Гість:** окремий потік `POST /api/v1/auth/guest` — серверна сесія; деталі в OpenAPI.
4. Коди помилок і тіла відповідей — у [openapi.yaml](/docs/openapi.yaml).

## Чекліст розділів для `/llms.txt` (llms.txt proposal)

- [x] `# Title` — назва продукту
- [x] `> summary` — короткий зміст
- [x] `## Docs` — посилання на OpenAPI, специфікацію, цей файл
- [x] `## Public HTTP` — анонімні GET
- [x] `## Auth` — cookie / CSRF / гість (без секретів)
- [x] `## Examples` — типові префікси API
- [x] `## Optional` — клон репозиторію, обмеження

## Сценарії (логічні кроки, не скрипти)

**Перегляд вітальні (публічно):** `GET /api/v1/landing` → відобразити тексти/прапорці, дозволені відповіддю.

**Реєстрація → чат:** `POST .../auth/register` → `POST .../auth/login` або одразу логін → `GET .../rooms` → `GET .../rooms/{id}/messages` → `POST .../rooms/{id}/messages` (з CSRF для браузерного клієнта).

**Гість:** `POST .../auth/guest` → далі обмежений набір дій (див. ролі в OpenAPI).

## Що LLM «не бачить» без JS

- Фактичний рендер UI та порядок завантаження клієнтських модулів.
- Підписки на real-time без спочатку отриманої сесії та кроку авторизації broadcast-каналу (шлях у OpenAPI).
- Внутрішній клієнтський стан — орієнтуйтесь на **відповіді API**.

**Компенсація:** [OpenAPI](/docs/openapi.yaml), цей файл, `/llms.txt`; глибокі схеми БД і операційні runbook — лише в **приватному** клоні репозиторію, не в публічному індексі.

## `robots.txt` та AI-краулери

Поведінка пошукових і AI-ботів **не** зафіксована в цій задачі як обов’язкова. За потреби продукт додає правила окремо (узгодити з юридичною політикою).

## Де шукати деталі далі

- **Публічно на цьому хості:** лише [OpenAPI](/docs/openapi.yaml) та посилання з `/llms.txt` / цього файлу.
- **Повний репозиторій** (структура legacy, QA-нотатки, runbook): за доступом у команди продукту; **не** вважати публічним контекстом для загальних LLM без явного дозволу.

Оновлюйте цей файл при суттєвих змінах публічних флоу або політики для агентів.
