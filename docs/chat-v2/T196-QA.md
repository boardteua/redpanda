# T196 — QA: ігнор користувачів у стрічці та архіві

**Статус:** PASS (2026-04-01).

## Аудит (очікування → факт → шар)

| Сценарій | Очікування | Було | Стало |
|----------|------------|------|--------|
| Історія кімнати `GET …/rooms/{id}/messages` | Не показувати `public` / `inline_private` від ігнорованих авторів | Фільтра не було | `IgnoredRoomMessageVisibility` у `RoomMessageHistoryQuery` |
| Архів `GET …/archive/messages` | Те саме для публічних рядків у доступних кімнатах | Фільтра не було | Той самий scope у `ChatArchiveController` |
| WebSocket `MessagePosted` / `RoomInlinePrivatePosted` | Клієнт не додає рядки ігнорованих | Усі рядки мерджились | `shouldHideRoomMessageDueToIgnore` у `mergeMessage` |
| Додати ігнор (меню / API) | Прибрати вже завантажені рядки зі стрічки | Залишались до F5 | `pruneIgnoredMessagesFromFeed` після додавання та після `/ignore` |
| Зняти ігнор | Знову бачити рядки автора | Лише після ручного оновлення | `removeIgnore` → `loadMessages()` |
| `/ignoreclear` | Повна стрічка знову | Частково | `loadMessages()` після slash |
| Приват / взаємний блок | Вже було | `PrivateMessageGate` | без змін |

## Автотести

- `php artisan test tests/Feature/RoomMessageIgnoreFilterTest.php`
- `php artisan test tests/Feature/ChatArchiveApiTest.php --filter=archive_excludes_public_messages_from_ignored`

## Ручний чекліст (2 акаунти)

1. B пише в кімнату; A додає B в ігнор — повідомлення B зникають зі стрічки A; B і третій користувач бачать B.
2. A знімає ігнор — після оновлення стрічки з’являються минулі публічні рядки B (через `loadMessages`).
3. Приват між A і B при взаємному ігнорі — як раніше 403 (існуючі тести).
