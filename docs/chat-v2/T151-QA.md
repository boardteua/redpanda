**Статус: PASS** (2026-03-26)

## Автотести

- `cd backend && php artisan test` — **PASS** (у т.ч. `Tests\Feature\SystemBotApiTest`: welcome ідемпотентно; після повторного `GET …/messages` **немає** `room_join`; `POST /api/v1/chat/system-bot/avatar` — 403 не-адмін / 201 адмін).
- `cd backend && npm run build` — **PASS**.

## Поведінка (коротко)

| Вимога | Реалізація |
|--------|------------|
| Аватар бота в адмін-UI | Вкладка «Руда панда» у `ChatSettingsModal.vue`: прев’ю + `POST /api/v1/chat/system-bot/avatar` (multipart `image`). |
| Фон рядка в стрічці | Клас `rp-chat-feed-row--system` і стилі з partial прибрані; системні пости використовують ту саму смугастість, що й звичайні рядки. |
| Join / F5 | Якщо в `chat_bot_welcome_sent` вже є `(user_id, room_id)`, **не** викликати `room_join` при наступних завантаженнях історії (`RedPandaBotRoomOpenTriggers`). Кеш `CHAT_BOT_JOIN_DEBOUNCE_SECONDS` **видалено** з `config/chat.php`. |

## OpenAPI

- `docs/chat-v2/openapi.yaml` — **0.18.2**, шлях `POST /api/v1/chat/system-bot/avatar`.

## Ручна перевірка (опційно для оператора)

- Адмін: вкладка бота → завантажити аватар → у стрічці нові пости бота з оновленим зображенням (існуючі рядки зберігають знімок `avatar` на момент посту).
- F5 у кімнаті після welcome: новий рядок «зайшов» не з’являється.
