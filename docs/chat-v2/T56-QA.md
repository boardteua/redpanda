# T56 — QA: індикатор непрочитаних приватних повідомлень

## Семантика

- **Непрочитане** — лише **вхідні** повідомлення в парі (від `peer` до поточного користувача), з `id` більшим за збережений «останній прочитаний вхідний» для цієї пари.
- **Прочитано:** після успішного `GET /api/v1/private/peers/{peer}/messages` або `POST /api/v1/private/peers/{peer}/read` бекенд виставляє watermark до поточного `MAX(id)` вхідних від цього peer.
- **Таб-badge:** сума непрочитаних по всіх тредах = `meta.total_private_unread`; на рядку розмови — `unread_count`.

## Автоматичні перевірки

- `php artisan test` — у т.ч. `PrivateMessageApiTest` (лічильники, зняття непрочитаного після `GET` треду та після `POST …/read`).
- `npm run build` — без помилок.

## Ручний сценарій (опційно)

1. Два акаунти: A надсилає B приват, B не відкривав тред — у B на вкладці «Приват» червоний badge з лічильником.
2. B відкриває тред з A — badge зменшується/зникає після оновлення списку розмов.
3. Нове вхідне через WS, тред з відправником відкритий — повідомлення з’являється, лічильник не росте (виклик `POST …/read`).
