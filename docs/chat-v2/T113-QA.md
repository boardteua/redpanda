# T113 — QA: legacy users з публічним чатом + rsync аватарок

**Задача:** [project-tasks/chat-v2-tasklist.md](../../project-tasks/chat-v2-tasklist.md) — T113.

## Зміни

- `LegacyBoardImportService`: імпорт лише **legacy.users** з **≥1** рядком у **legacy.chat**; звіт **`users_legacy_total`**, **`users_skipped_no_posts`**.
- `LegacyImportUserSelection`: чиста логіка фільтра (unit-тест).
- `chat:legacy-inspect`: рядок **legacy.users без chat**.
- `chat:legacy-sync-avatars`: обгортка **rsync** + SSH (`BatchMode`), `--dry-run` → `-n`.
- `config/legacy.php`, `backend/.env.example`, документація **T113-LEGACY-AVATARS.md**, оновлення **T13-ETL-STAGING.md**.

## Автоматичні перевірки

- `cd backend && php artisan test` — PASS (**328** tests; `LegacyImportUserSelectionTest`, `ChatLegacyCommandsTest::test_legacy_sync_avatars_fails_when_not_configured`).

## Ручний чек (оператор)

1. На staging з реальною legacy БД: `php artisan chat:legacy-inspect` — число «без публічних постів» узгоджується з очікуванням.
2. `php artisan chat:legacy-import-staging --dry-run` — **користувачі для імпорту** + **пропущено** узгоджені з inspect.
3. Заповнені `LEGACY_AVATAR_RSYNC_*`: `php artisan chat:legacy-sync-avatars --dry-run` — вивід rsync без помилок SSH (ключ у агенті).
