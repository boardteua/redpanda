# T09 — Архів чату (QA)

**Задача:** табличний архів з пагінацією та пошуком (паритет [SITE-STRUCTURE §5](../board-te-ua/SITE-STRUCTURE.md)).

## Автоматичні перевірки

```bash
cd backend && php artisan test --filter=ChatArchiveApiTest
cd backend && php artisan test --filter=SpaShellTest
cd backend && npm run build
```

Очікування: усі тести **PASS**, збірка без помилок.

## API

| Метод | Шлях | Примітки |
|-------|------|----------|
| GET | `/api/v1/archive/messages` | `auth:sanctum`, throttle **`archive-read`** (60/хв/користувач) |
| Query | `page` ≥ 1 | За замовчуванням 1 |
| Query | `per_page` | Лише **10, 25, 50, 100** (дефолт 25) |
| Query | `q` | До 200 символів; LIKE по `post_message` та `post_user` |
| Query | `room` | Опційно `room_id`; 403 якщо немає `interact`, 404 якщо кімнати немає |

Область даних: усі повідомлення з кімнат, доступних користувачу за `RoomPolicy::interact` (гість — лише `access === 0`).

## SPA

| Маршрут | Компонент |
|---------|-----------|
| `/archive` | `ArchiveChat.vue` |

- Вибір «дописів на сторінку»: 10 / 25 / 50 / 100.
- Таблиця: **Користувач** | **Повідомлення** | **Дата**.
- Пагінація: Перша / Попередня / Наступна / Остання + текст «Сторінка X з Y».
- З `/chat`: посилання **Архів чату** передає `?room=` поточної кімнати (фільтр у архіві).
- З головної після входу: `/?history=1` → редірект на **`/archive`** (аналог `?history=1` на board.te).

## Ручний чекліст (браузер)

1. Увійти → **Архів чату** з головної або з чату.
2. Змінити per_page, переконатися, що таблиця та лічильники оновлюються.
3. Ввести пошук, **Шукати** (або Enter) — відфільтровані рядки.
4. Пагінація на другу сторінку при достатній кількості повідомлень.
5. Гість: переконатися, що в архіві немає дописів із кімнат «тільки для зареєстрованих» (якщо такі є в БД).

## Продуктивність (staging / за потреби)

Пошук використовує `LIKE '%…%'` — на великих таблицях варто перевірити `EXPLAIN` і навантаження; індекси під повнотекст — окреме покращення поза мінімальним T09.
