# T118 — QA: Threads (офіційний oEmbed) + TikTok short links

## Посилання

- [Embed a Threads Post](https://developers.facebook.com/docs/threads/tools-and-resources/embed-a-threads-post/) (Meta)
- [Threads oEmbed reference](https://developers.facebook.com/docs/threads/reference/oembed) (параметри `url`, `maxwidth` 320–658; `maxheight` не підтримується)

## Ланцюжок (permalink → UI)

1. У тексті повідомлення збігається permalink поста Threads (`www.threads.com` / `threads.com` / `threads.net`): шлях `/@…/post/…` або `/t/{shortcode}`.
2. Парсер емітить сегмент **`oembedPending`** з **точним** `href` цього URL (у одному повідомленні кілька URL → кілька незалежних сегментів; TikTok short link не підміняє Threads).
3. `ChatOembedBlock` викликає **`GET /api/v1/oembed?url=…`** (T55, Sanctum).
4. Бекенд за реєстром `oembed-providers.json` шле запит лише на **`https://graph.threads.net/v1.0/oembed`** з `format=json`, **`omitscript=true`**, **`maxwidth`** у діапазоні 320–658 (кламп у `OEmbedProxyService`).
5. Поле **`html`** санітизується (без `<script>`, без `on*`, обмеження посилань на домени Threads); iframe-шлях для Threads **не** використовується.
6. Клієнт ренерить санітизований blockquote (**`v-html`**, довіряємо лише відповіді нашого API), один раз підключає **`https://www.threads.com/embed.js`**, викликає **`window.instgrm.Embeds.process()`** після монтування (динамічний SPA).
7. При помилці / порожньому `html` — **fallback** як звичайне посилання (як і для інших oEmbed).

## TikTok (`vm.tiktok.com` / `vt.tiktok.com`)

У реєстрі додані схеми short links; запит іде на **`https://www.tiktok.com/oembed`** з оригінальним `url`. Якщо апстрім повертає помилку (редиректи з вихідного клієнта вимкнені — T55), клієнт показує **посилання** без порожнього «битого» блоку.

## Обмеження та відповідність Meta

- Лише **публічні** пости; **~1000 запитів/год** на oEmbed — відповідь кешується на бекенді (TTL з `cache_age` або дефолт).
- Метадані та контент з oEmbed — **лише для відображення embed у UI**, без збору аналітики з цих даних (умови Meta).

## Ручна перевірка (опційно)

- Публічний permalink Threads у повідомленні → у стрічці видно rich-картку після завантаження `embed.js` (або fallback-посилання).
- `curl` до `/api/v1/oembed?url=<threads permalink>` з сесією (див. `T55-oembed.md`) — JSON з `type":"rich"`, `html` без `<script>`.
