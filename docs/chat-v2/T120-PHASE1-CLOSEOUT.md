### Backend (`backend/app`)

- **Структура:** збережено шар Policies / Requests / Resources / Events; після T106 критичний шлях PATCH/DELETE повідомлень проходить через сервіс — позитивно для тестованості.
- **`ChatMessageController`:** орієнтовно **~444** рядки (було ~470 у T99) — частина логіки винесена; подальше дроблення за доменом лишається **P2** backlog, не блокер фази 1.
- **N+1 / продуктивність:** пункт T99 про агрегований unread для привату **закритий** (T102). Рекомендація T99 щодо вартості Gate у списках **адресована** батчем (T105).
- **OpenAPI ↔ маршрути:** після T101 основний розрив T99 усунуто; для фази 2 варто періодично проганяти звірку при додаванні нових `api.php` шляхів (ручний diff або lint).

### Frontend (`backend/resources/js`)

- **Архітектура:** Vuex досі не використовується для кімнати; стан і Echo зосереджені в `ChatRoom.vue`, але обсяг зменшено за рахунок модулів у `js/chat/` і розбиття сайдбара (**T103–T104**).
- **Echo / real-time:** подальші зміни лишаються під регресією WS (**T11**, **T80**, **T96-QA** за трасуванням у чеклісті).
- **Великі SFC:** `ChatRoom.vue` лишається важким файлом — **P1** на фазу 2: цільова декомпозиція (composables/mixins за доменом) окремими Txx без зміни поведінки.

### Залежності та безпека пакетів

- **Composer (`composer audit`, 2026-03-25):** `advisories: []`, `abandoned: []`.
- **npm (`npm audit`, 2026-03-25):** **3 × low**, пов’язані з **Vue 2.7** (ReDoS у `parseHTML`, GHSA-5j4c-8p2g-v4jx) та транзитивно з `@vitejs/plugin-vue2`, `@linusborg/vue-simple-portal`; **fixAvailable: false** у межах Vue 2 — прийнятний **відомий** ризик до можливої міграції на Vue 3; деталі — **T107** / періодичний audit у CI.
