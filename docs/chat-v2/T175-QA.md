# T175 — ProxyCheck anti-spam — QA

**Status:** PASS (2026-03-31)  
**Зміна:** server-side інтеграція `proxycheck.io` (v2) на auth + posting, з кешем і graceful-degradation; фіксація `ban_evasion_events` для banned IP та proxycheck-deny.

## Автоматичні перевірки

| Крок | Команда | Результат |
|------|---------|-----------|
| ProxyCheck feature tests | `cd backend && php artisan test tests/Feature/ProxyCheckAntiAbuseTest.php` | PASS |
| Impacted tests (auth/chat/private) | `cd backend && php artisan test tests/Feature/AuthApiTest.php tests/Feature/ChatApiTest.php tests/Feature/PrivateMessageApiTest.php` | PASS |

## Ручні сценарії (smoke)

1. **Proxy/VPN deny:** увімкнути `PROXYCHECK_ENABLED=true`, задати `PROXYCHECK_API_KEY`, відправити запит на:
   - `POST /api/v1/auth/register` або `POST /api/v1/rooms/{room}/messages`
   - очікування: **403** + `code=proxycheck_denied`
2. **Outage path:** тимчасово заблокувати доступ бекенда до `proxycheck.io` (або виставити дуже малий timeout) і повторити реєстрацію/постинг
   - очікування: продукт **не ламається**, запит проходить (graceful degradation), у логах є warning.
