# T10 — Медіа в чаті (QA)

**Задача:** завантаження зображень, таблиця `images`, обмеження MIME/розміру, прикріплення до публічного повідомлення; файли поза webroot; зрозумілі помилки для UI.

## Автоматичні перевірки

```bash
cd backend && php artisan test --filter=ChatImageApiTest
cd backend && php artisan test --filter=DatabaseSchemaTest
cd backend && npm run build
```

Очікування: усі тести **PASS**, збірка без помилок.

## API

| Метод | Шлях | Примітки |
|-------|------|----------|
| POST | `/api/v1/images` | `auth:sanctum`, CSRF на змінюючих; `multipart/form-data`, поле **`image`**; MIME `image/jpeg`, `image/png`, `image/gif`, `image/webp`; max **4096 КБ**; throttle **`image-upload`** (20/хв/користувач) |
| GET | `/api/v1/images/{id}/file` | Потік байтів; доступ: власник **або** повідомлення з `file = id` у кімнаті, де є `interact`; throttle **`image-read`** (180/хв/користувач) |
| POST | `/api/v1/rooms/{roomId}/messages` | Опційно **`image_id`** (власне зображення після upload); потрібен непорожній `message` (після trim) **або** `image_id` |

Відповіді з повідомленнями містять **`file`** (0 або `images.id`) та **`image`** `{ id, url }` або `null`.

## SPA

| Маршрут | Компонент |
|---------|-----------|
| `/chat` | `ChatRoom.vue` — вибір файлу, прев’ю, відправка з `image_id`; рендер `<img>` у стрічці |
| `/archive` | `ArchiveChat.vue` — міні-прев’ю, якщо є вкладення |

## Ручний чекліст (браузер)

1. Увійти → `/chat`, обрати кімнату.
2. Прикріпити JPEG/PNG ≤ 4 МБ, переконатися в прев’ю, надіслати (можна лише зображення без тексту).
3. Перевірити відображення в стрічці та (за наявності) в архіві для тієї ж кімнати.
4. Спроба завантажити неприпустимий тип або завеликий файл — очікується **422** з полями помилок.
5. Другий користувач у тій самій кімнаті бачить зображення в стрічці (URL з авторизацією cookie).
