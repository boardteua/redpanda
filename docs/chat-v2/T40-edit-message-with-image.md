# T40 — Редагування повідомлення з зображенням (політика)

## Рішення

- **PATCH** `/api/v1/rooms/{roomId}/messages/{postId}` змінює лише **текст** (`message`) та **стиль** (`style`). Поле **`chat.file` / вкладене зображення не змінюється** цим ендпоінтом.
- Порожній текст після `trim` **дозволений**, якщо в повідомленні вже є вкладення (`file > 0`) — валідація в `UpdateChatMessageRequest`.
- **Заміна або від’єднання** зображення під час редагування **не підтримуються** (немає полів у PATCH). Щоб змінити картинку — **новий допис** або **видалення** повідомлення (T37), потім за потреби новий текст + upload.
- **UI композера** у режимі редагування: показує прев’ю поточного зображення і коротке пояснення; кнопки додавання зображення, бібліотека «мої зображення» та paste-картинки **вимкнені** (як і раніше).

## Паритет / спека

У `docs/board-te-ua/` окремої норми «редагувати вкладення» не знайдено; обрано мінімальний безпечний варіант без розширення контракту API.

## Echo

Подія `MessageUpdated` містить `file` та `image` (url), якщо вкладення є — інші клієнти не втрачають прев’ю після редагування тексту.
