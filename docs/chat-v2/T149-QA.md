## Відповіді на відкриті питання з чекліста

| Питання | Рішення |
|--------|---------|
| У **яку** кімнату постити оголошення про нову публічну кімнату | За замовчуванням — **найменший `room_id`** серед `access = public`, або явно **`CHAT_BOT_ANNOUNCE_ROOM_ID`** у `.env` / `config/chat.php`. |
| Шаблони / тумблери в опціях чату | **Не** входить у T149; тексти — `lang/uk/chat.php`; керування з адмінки можна винести в наступний таск. |
| Дебаунс join | **`CHAT_BOT_JOIN_DEBOUNCE_SECONDS`** (дефолт **90** с), ключ кешу `rp_bot_join:{user_id}:{room_id}`. |
| Згадки / звук (T123) | `type === system` не обробляється префіксом «нік >»; **`mentioned_user_ids`** порожній. |

## Контракт для T150

- REST і **Echo `MessagePosted`**: для `type: system` додано **`system_kind`**, **`target_room_id`**, **`action_label`** (див. `docs/chat-v2/openapi.yaml` → `ChatMessage`).
- Профіль бота: **`GET|PATCH /api/v1/chat/system-bot/profile`** (лише `chat-admin`; `GET` — для форми в T150).

## Примітка щодо ніку бота

Сидер створює нік **«Руда панда»** (з пробілом). **`PATCH`** профілю бота вимагає той самий regex, що й реєстрація (**без пробілів**); для зміни ніку через API використовуйте варіант без пробілів або оновіть рядок у БД вручну на staging.
