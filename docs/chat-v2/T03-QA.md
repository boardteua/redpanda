# T03 — QA (Vue: логін / реєстрація / гість)

**Вердикт:** PASS (автоматично: `npm run build`, `php artisan test`; візуально — чекліст нижче).

## Скріншоти (додати вручну в репозиторій за потреби)

Рекомендовані кадри для доказу UI:

1. `/` — режим **Вхід**, світла тема.
2. `/` — режим **Реєстрація**, поле email видно.
3. `/` — темна тема (після циклу кнопки теми).
4. Після успішного входу — екран «Ви в чаті».
5. Помилка валідації (422) — підсвічені поля / `role="alert"`.

Шлях для збереження (пропозиція): `docs/chat-v2/screenshots/t03/` (створити при зйомці).

## Клавіатурна навігація (чекліст)

У браузері без миші (лише Tab / Shift+Tab / Enter / Space):

- [ ] З фокусу адресного рядка: Tab потрапляє на кнопку **теми**, далі на таби **Вхід** / **Реєстрація**, далі по полях форми, кнопку submit, блок гостя.
- [ ] **Enter** на активному табі перемикає режим (або використати Space для кнопок, де застосовно).
- [ ] У режимі **Реєстрація** порядок табів логічний: нік → email → пароль → підтвердження → кнопка.
- [ ] Видиме **focus-visible** кільце на інтерактивах (кнопки, поля, таби, посилання-стиль).
- [ ] Після помилки сервера оголошення з `role="alert"` читається скрінрідером (перевірка за бажанням у VoiceOver / NVDA).

## Технічні нотатки

- Sanctum SPA: перед `POST` на `/api/v1/auth/*` викликається `GET /sanctum/csrf-cookie`; axios: `withCredentials` + `withXSRFToken` у `resources/js/bootstrap.js`.
- Тема: `data-theme` на `<html>` — `system` | `light` | `dark`, ключ LocalStorage `redpanda-theme`.
- Токени кольору: `resources/css/app.css` (змінні `--rp-*`), контраст форм орієнтовно під **WCAG AA** для тексту та кнопки primary.
