# T119 — мобільна кімната чату (QA)

## Зміни (каркас + композер)

- **`ChatRoom.vue`:** корінь `/chat` — **`min-h-0 overflow-hidden`**; на вузькому екрані або **`100dvh`** (fallback), або **рамка з `visualViewport`** (див. нижче); на **`md+`** — **`100dvh`** як раніше.
- **`rp-chat-external-wrap`:** на `max-md` додано **`min-h-0 overflow-hidden`** для коректного стискання flex-дітей.
- **`ChatRoomMainColumn.vue`:** **`overflow-hidden`** на всіх breakpoints (раніше лише `md:`), щоб середня колонка не випускала скрол назовні.
- **`app.css` + `ChatRoomComposer.vue`:** на **`max-width: 767px`** зменшено **`min-height`** поля вводу до **~однієї лінії** (`2.25rem` у CSS, `minPx = 36` у `syncComposerInputHeight`); багаторядковий ввід лишається з **`max-height` / внутрішнім скролом** textarea.
- **iOS Safari + клавіатура:** корінь `/chat` на `max-width: 767px` з **`visualViewport`** (`position: fixed`, `top` = `offsetTop`, `height` = `height`) — прибирає «дірку» між композером і input accessory / клавіатурою. Нижній `safe-area` padding на формі композера прибрано, щоб не дублювати відступ поверх уже зменшеного vv.

## Чекліст ручної перевірки

1. DevTools / девайс ≤767px: вертикальний скрол **лише** у списку повідомлень; хедер і композер **не** «плавають» посередині при скролі сторінки (сторінка не скролиться).
2. Порожнє поле вводу — візуально **одна лінія**; Shift+Enter — кілька рядків до ліміту, далі скрол усередині textarea.
3. Відкрити off-canvas сайдбар — після закриття лейаут кімнати без зламаних висот.
4. **iPhone Safari:** фокус у полі повідомлення — композер **без великого порожнього зазору** над панеллю клавіатури / autofill.
5. **`npm run build`** — PASS (артефакт збірки).
