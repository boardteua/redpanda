# T119 — мобільна кімната чату (QA)

## Зміни (каркас + композер)

- **`ChatRoom.vue`:** корінь `/chat` — **`100dvh` + `min-h-0` + `overflow-hidden`** (без JS layout); **`md+`** — те саме в рядку з flex.
- **`spa.blade.php`:** у viewport meta додано **`interactive-widget=resizes-content`** (MDN) — браузер змінює layout viewport під клавіатуру **без** `visualViewport` у Vue.
- **`rp-chat-external-wrap`:** на `max-md` — **`min-h-0 overflow-hidden`**.
- **`ChatRoomMainColumn.vue`:** **`overflow-hidden`** на всіх breakpoints.
- **`app.css`:** мобільний композер — **`font-size: 1rem`** у полі вводу (уникнення автозуму iOS при фокусі); **`min-width: 0` / `overflow-x: hidden`** на рядку композера; **`overscroll-behavior-y: contain`** на стрічці (`ul[role='log']`); **`color-scheme`** на `html` за темою; кнопка **«Надіслати»** — **`appearance: none`**, **`background-color`**, токени **`--rp-chat-send-*`** узгоджені з **`--rp-primary`** і в світлій темі.
- **`ChatRoomComposer.vue`:** форма й рядок — **`min-w-0 w-full max-w-full`**.
- **`syncComposerInputHeight`:** лише висота textarea (не layout viewport).

## Чекліст ручної перевірки

1. DevTools / девайс ≤767px: вертикальний скрол **лише** у списку повідомлень; хедер і композер **не** «плавають» посередині при скролі сторінки (сторінка не скролиться).
2. Порожнє поле вводу — візуально **одна лінія**; Shift+Enter — кілька рядків до ліміту, далі скрол усередині textarea.
3. Відкрити off-canvas сайдбар — після закриття лейаут кімнати без зламаних висот.
4. **iPhone Safari:** фокус у полі — **без стрибка масштабу**; кнопка надсилання **видима** і в **темній** темі; скрол історії **без підкручування** всієї сторінки.
5. **`npm run build`** — PASS (артефакт збірки).
