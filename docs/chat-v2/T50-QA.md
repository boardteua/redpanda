# T50 — QA (Друзі: список обраних незалежно від онлайн)

## Результат

**PASS** (автоматичні перевірки в CI-стилі локально).

## Доказ

1. **Бекенд:** `GET /api/v1/friends` уже повертав усіх прийнятих друзів без фільтра за присутністю; додано `DELETE /api/v1/friends/{user}` для прибирання з друзів / скасування вихідного запиту.
2. **Тести:** `php artisan test --filter=FriendApiTest` — усі кейси зелені, включно з `test_destroy_*`.
3. **Фронт:** вкладка «Мої друзі» (колишній підпис «Активний») показує повний список; індикатор статусу для друга без даних у кеші кімнати трактується як неактивний (сірість), без приховування рядка; після дій з меню (запит, прийняття, видалення) відкривається вкладка «Друзі» з відповідним підвкладенням.
4. **Збірка:** `npm run build` у каталозі `backend/` — без помилок.
5. **OpenAPI (доповнення):** `paths./api/v1/friends/{userId}` `delete` + схема `ApiPlainMessageResponse` (версія специфікації 0.16.2).

## Примітки для ручної перевірки оператором

- Додати друга з меню «Люди» → відкривається «Друзі» / «Запити на дружбу»; після прийняття — рядок у «Мої друзі» лишається, якщо друг офлайн (у тій самій кімнаті статус оновлюється з `presence-statuses` / WS).
