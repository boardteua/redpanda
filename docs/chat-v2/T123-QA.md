# T123 — QA: звук згадки (mentions) у кімнаті

## Вердикт

**PASS** (після `npm run build` у `backend/` та релевантних PHPUnit).

## Джерело правди для «згадано мене»

Сервер обчислює **`mentioned_user_ids`** для повідомлень з `type === 'public'`, якщо тіло після trim відповідає шаблону **«нік > …»** (пробіли навколо `>` як у T25). Нік резолвиться в `users.user_name` **без урахування регістру**. Поле є в **REST** (`ChatMessageResource`) і в **broadcast** події **`MessagePosted`**.

## Звук

- **`/sounds/mention.mp3`** — окремий файл (каталог `public/sounds` у `.gitignore`, як **`newpost.mp3`**). Для локальної перевірки скопіювати наприклад `newpost.mp3` → `mention.mp3`.
- **`notification_sound_prefs.mentions`** — якщо **вимкнено**, звук згадки **не** відтворюється.
- **`volume_percent`** — спільно з іншими чат-звуками через `playChatNotificationSound`.
- **Не від себе**, **silent_mode**, **inline_private** / **client_only** — ті самі винятки, що для кімнатного newpost (T65).
- **Видимість вкладки** — як T65: без **sound_on_every_post** (T75) звук згадки **лише** при `document.visibilityState === 'visible'`.
- **Пріоритет (дедуп):** якщо одночасно підходить і згадка, і newpost — відтворюється **лише mention**.

## Ручний чекліст

1. Два акаунти, Reverb + queue, **активна** кімната, вкладка на передньому плані: користувач A надсилає **`B'sNick > привіт`** — B чує **mention** (якщо канал згадок увімкнено).
2. Те саме без префікса — B чує лише **newpost** (якщо увімкнено «звичайні повідомлення»).
3. У профілі B вимкнути **«Згадки (mentions)»** — **mention** не грає; newpost лишається за правилами T65.
4. Вкладка у **фоні**, без legacy «звук на кожен пост» — **ні** mention, **ні** newpost.

## Трасування

Закриває відкрите питання T65 про окремий звук для згадок; узгоджено з T24/T25.
