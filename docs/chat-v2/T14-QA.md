# T14 — Інтеграція (QA)

**Задача:** підтвердити скрізний проєкт «логін → чат → сайдбар → приват (T08)»; reality check перед завершенням обов’язкового чеклісту.

## Автоматичні докази (API + оболонка)

```bash
cd backend && php artisan test --filter=IntegrationFlowApiTest
cd backend && php artisan test
cd backend && npm run build
```

- **`IntegrationFlowApiTest`** — зареєстрований користувач: кімнати (включно з закритою), історія, POST повідомлення, архів, приватні розмови, друзі, ігнор.
- **Гість:** лише публічна кімната + успішний POST у неї.
- **`SpaShellTest`** — `/`, `/chat`, `/archive` віддають Vue-оболонку.

Це **не** замінює повну ручну перевірку UI (Echo, кліки, мобільний off-canvas).

## Ручний чекліст (браузер)

Перед релізом або після великих змін у фронті пройти послідовно:

1. **Головна `/`** — вхід або реєстрація або гість; тема / форми без критичних a11y регресій (див. T03-QA).
2. **`/chat`** — стрічка оновлюється після відправки; за наявності Reverb — live з іншого сеансу; без WS — poll / банер degraded (T06-QA).
3. **Сайдбар 320px** — вкладки Люди, Кімнати, Друзі, Приват, Ігнор; порожні стани; на вузькому екрані off-canvas (T07-QA).
4. **Приват** — відкриття з меню / адресація, відправка, отримання (T08-QA).
5. **`/archive`** — пагінація, пошук, фільтр кімнати з чату за `?room=` (T09-QA).
6. **Медіа** — прикріпити зображення, побачити в стрічці (T10-QA).
7. **Модерація** — лише API; UI панелі — окремий етап (T12-QA).
