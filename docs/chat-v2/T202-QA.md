# T202 — QA: бейдж вхідних запитів на вкладці «Друзі»

## Автоматично

- `cd backend && npm run build` — PASS (зафіксовано під час закриття задачі).

## Ручний сценарій (два акаунти)

1. Увійти як **A** і **B** (два браузери / вікна інкогніто).
2. **B** надсилає запит на дружбу **A**.
3. У **A**: на вкладці «Друзі» (`#chat-tab-d-friends` на desktop, `#chat-tab-m-friends` у мобільному ряді табів) з’являється червоний бейдж з кількістю (cap як у приватів: `99+` після 99).
4. Відкрити вкладку «Друзі» → прийняти або відхилити запит → бейдж зникає або число оновлюється без повного перезавантаження сторінки (після `loadFriendsAndIgnores` / Echo — згідно з T08).

## Регресія

- **T56:** вкладка «Приват» — бейдж і `aria-label` без змін у поведінці.
- **T168:** мобільна кнопка меню в шапці — без змін (T202 не додає бейдж на бургер, лише на таби сайдбару за специфікацією задачі).

## Примітка

- `aria-label` для таба «Друзі» з очікуючими запитами: «Друзі, 1 вхідний запит на дружбу» / «Друзі, N вхідних запитів на дружбу».
