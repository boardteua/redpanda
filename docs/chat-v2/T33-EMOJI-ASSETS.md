# T33 — очікуваний формат пакета смайлів (асети)

Після того як клієнт додасть бібліотеку «колобків» (GIF/SVG), можна підключити її без зміни контракту вставки в поле: **`:` + `code` + `:`** (як у legacy [CHAT-MAIN-INPUT.md](../board-te-ua/CHAT-MAIN-INPUT.md)).

## Рекомендований manifest (JSON)

Файл на CDN або в `public/` (без великих бінарників у git за політикою репо):

```json
{
  "version": 1,
  "baseUrl": "/emoticon/",
  "items": [
    { "code": "hi", "title": "Привіт", "file": "hi.gif", "keywords": ["hello", "привіт"] }
  ]
}
```

- **`code`** — унікальний ідентифікатор; у текст повідомлення вставляється `:code:`.
- **`title`** — короткий підпис для пошуку та `title` на кнопці.
- **`file`** — ім’я файлу відносно `baseUrl` (або повний URL, якщо узгоджено).
- **`keywords`** — опційно, додаткові рядки для фільтра пошуку.

## UI

- Модал показує сітку кнопок; для кожного елемента — прев’ю (`<img>` або emoji-плейсхолдер) і підпис `:code:`.
- Пошик фільтрує за `code`, `title` та `keywords` (регістронезалежно).

## Рендер у стрічці

Підстановка зображень замість `:code:` у відображенні повідомлень — окремий таск/ітерація (потрібен санітизований HTML або окремий компонент рядка).
