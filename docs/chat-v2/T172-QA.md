# T172 — Web Push: аватар автора в системному сповіщенні — QA

**Статус:** PASS (2026-03-30)  
**Зміна:** Підписаний публічний URL аватара (`GET /api/v1/public/users/{user}/avatar`, middleware `signed`) — ОС може завантажити зображення без Sanctum-cookies; у JSON push-пейлоад додано `author_avatar_url`; `icon` на бекенді збігається з аватаром або PWA fallback; `sw.js` використовує `author_avatar_url` для `icon`, опційно `image` для прев’ю.

## Контракт пейлоаду (бекенд → SW)

| Поле | Опис |
|------|------|
| `author_avatar_url` | `string \| null` — абсолютний URL (тимчасово підписаний), якщо у автора є `avatar_image_id` |
| `icon` | Те саме, що видимий малий аватар, або `/pwa/icon-192.png` через `url()` |
| `badge` | Без змін: іконка додатку (`icon-96`) |

TTL підпису: `config('services.web_push.public_avatar_url_ttl_seconds')` (дефолт 604800 с; перевизначення: `WEB_PUSH_PUBLIC_AVATAR_TTL_SECONDS`).

## Ручний smoke (HTTPS, без PII у скріншотах)

1. Два зареєстровані користувачі A і B; у A є завантажений аватар; B підписаний на Web Push.  
2. **Кімната:** A пише в кімнаті → у B push з іконкою аватара A (або великим прев’ю там, де ОС підтримує `image`).  
3. **Приват:** A пише B → те саме, іконка A.  
4. Користувач без аватара → іконка PWA (**T164**), без «порожньої» картки.  
5. Клік по сповіщенні → deep link як у **T166** (регресія).

## Примітка

Повний `php artisan test` у локальному середовищі може давати фейли в тестах, жорстко прив’язаних до `http://localhost` / SEO-асетів (`SeoPublicRoutesTest`), якщо `APP_URL` або публічні шляхи відрізняються — це поза scope цієї задачі.
