# T181 — QA: VIP генерація зображень «Руда Панда»

**Дата:** 2026-04-01  
**Вердикт:** PASS

## Що зроблено

- Додано job **`GenerateRudaPandaVipImageJob`** (черга): **VIP-only** генерація зображення з текстового prompt.
- Виклик Gemini image generation через `generateContent` з `generationConfig.responseModalities = ["TEXT","IMAGE"]` (дока: [image generation](https://ai.google.dev/gemini-api/docs/image-generation)).
- Збереження зображення у disk **`chat_images`** + запис у таблицю **`images`** (той самий шлях, що й T10).
- Пост бота у таблицю `chat` з вкладенням: `chat.file = images.id`.
- Idempotency: `client_message_id` детерміновано з `IdempotencyKey` (`ruda-panda-image:<key>`), дубль не поститься (unique index).

## Автотести

```bash
cd backend && php artisan test --filter RudaPandaVipImageGenerationTest
```

Очікування: PASS (VIP створює `images` + `chat`, non‑VIP нічого не створює; `Storage::fake(chat_images)`, `Http::fake()`).

## Нотатки

- Перевірка прав зроблена **до** виклику Gemini: **не‑VIP не генерує** і не витрачає quota.
- Повна інтеграція “слухати кімнату та тригерити цей job по повідомленню” — у наступних задачах епіку; T181 дає готовий backend-шлях генерації та публікації.
