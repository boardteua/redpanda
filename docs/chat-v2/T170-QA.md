## Автоматичні перевірки

- `cd backend && npm run build` — PASS (2026-03-29).
- `cd backend && APP_URL=http://localhost php artisan test` — PASS, 427 tests (якщо в `.env` інший `APP_URL`, SEO-тести можуть очікувати `http://localhost/...`).

## Ручний сценарій (обов’язково для повного PASS)

1. Два акаунти; на пристрої A — PWA або Chrome з увімкненим Web Push.
2. Увійти в чат, відкрити кімнату X (або приват), згорнути застосунок / перевести вкладку у фон.
3. З акаунта B надіслати повідомлення в X (або в приват).
4. На A натиснути на push.
5. **Очікування:** відкривається чат; **текст повідомлення видно в стрічці** без оновлення сторінки.

**Платформа (заповнити при прогоні):** ___ (наприклад Android Chrome PWA / desktop installed PWA).

## Примітки

- Якщо Reverb / `VITE_REVERB_*` не налаштовані, push може приходити, а live-Echo — ні; цей фікс покладається на REST після `visibilitychange`, тож стрічка має зрівнятися з сервером після повернення у foreground.
