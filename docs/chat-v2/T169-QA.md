## Логіка приймання

| Стан | Очікування |
|------|------------|
| **granted** + активна підписка | Після F5 плашка не з’являється на кадр «ще не підключено» до `getSubscription` |
| **default** | Плашка з закликом з’являється без очікування SW (не блокуємо UX) |
| **denied** | Плашка прихована, як і раніше |
| **guest / !canShow** | Плашка не показується |

## Ручний smoke (браузер, PROD-збірка / staging)

1. Увімкнути push, перезавантажити сторінку (hard refresh ×2–3): не повинно бути короткого показу жовтої/закликальної плашки при вже наданому дозволі та валідній підписці.
2. Скинути дозвіл сайту → очікувана плашка для сценарію **default** / помилки scope тощо.
