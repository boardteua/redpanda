# T40 — QA: редагування повідомлення з зображенням

| Сценарій | Очікування |
|----------|------------|
| Відкрити редагування повідомлення **без** зображення | Банер «Редагування…» без підрядка про зображення; немає блоку прев’ю вкладення |
| Відкрити редагування повідомлення **з** зображенням | Банер з поясненням; прев’ю картинки; текст пояснює що вкладення не змінюється |
| Зберегти лише зміну тексту | PATCH 200; у стрічці той самий `image.url` / прев’ю |
| Очистити текст, залишити лише зображення (як у дописі) | Зберегти можна (`editHadFile`); API приймає порожній текст якщо `file > 0` |
| Paste / file picker / «Мої зображення» в режимі редагування | Недоступні або ігнорують upload (paste уже заблоковано) |
| Другий клієнт після чужого редагування | WS `MessageUpdated`: текст оновлено, зображення на місці |

## Команди

- `cd backend && php artisan test --filter=test_patch_message_with_image`
- `cd backend && npm run build`
