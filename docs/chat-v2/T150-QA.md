# T150 — QA: фронт «Руда панда» (стрічка, CTA, профіль бота)

**Статус: PASS** (закриття T150)

## Автоматичні перевірки

- `cd backend && npm run build` — **PASS**
- `cd backend && php artisan test --filter=SystemBotApiTest` — **PASS** (у т.ч. `GET /api/v1/chat/system-bot/profile` для адміна)
- `npx @apidevtools/swagger-cli validate docs/chat-v2/openapi.yaml` — **valid** (за наявності CLI)

## Реалізація (коротко)

- **Нормалізація повідомлень:** `normalizeMessage` зберігає `system_kind`, `target_room_id`, `action_label` для `type === system` (REST + Echo).
- **Стрічка:** рядок системного бота — клас `rp-chat-feed-row--system`, нік через `var(--rp-chat-nick-system)`; без кліку «згадати» / «приват» на боті.
- **CTA:** кнопка з `action_label` або «Перейти в кімнату» → `vue-router` `replace({ path: '/chat', query: { room } })` за `target_room_id`; приховано, якщо вже в цій кімнаті.
- **Звук:** `type === system` не запускає `newpost` / mention у `playActiveRoomIncomingSounds` (менше шуму на join/welcome).
- **Адмін:** вкладка «Руда панда» у `ChatSettingsModal` — `GET` + `PATCH` `/api/v1/chat/system-bot/profile` (нік, країна, «про мене»).

## Ручний сценарій (оператор)

1. Два браузери / кімната: системне повідомлення (welcome/join) — окремий вигляд рядка.
2. Створення публічної кімнати (VIP): у хабі оголошення з кнопкою переходу → відкривається нова кімната.
3. Адмін: налаштування чату → «Руда панда» → зміна ніку/опису → збереження → перезавантаження стрічки / новий пост від імені бота з оновленим ніком.
