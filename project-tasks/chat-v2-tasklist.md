# Chat v2 — task list (Agents Orchestrator)

**Правила:** одна задача в роботі; після реалізації — **код-рев’ю** (див. `docs/chat-v2/AGENT-ORCHESTRATION.md`), потім QA з **PASS** лише за доказом (логи, тести, скріншоти для UI). До **3** спроб на задачу; далі — escalate. Відкриті задачі: рядки `### [ ]`; закриті: `### [x]`.

**Специфікація:** [project-specs/chat-v2-setup.md](../project-specs/chat-v2-setup.md)  
**Оркестрація:** [docs/chat-v2/AGENT-ORCHESTRATION.md](../docs/chat-v2/AGENT-ORCHESTRATION.md)

### [x] T21 — Ролі чату (RBAC): адмін / модератор / VIP / зареєстрований / гість; політики, кімнати VIP, ліміти дописів

- **Delegate:** Backend Architect (+ коротко Frontend для полів відображення в API)
- **Deliverables:** універсальний **шар перевірки прав** на бекенді за найкращими практиками Laravel (узгодити в PR: **централізовані Policies/Gates + enum/константи ролей** vs пакет на кшталт Spatie Permission — з обґрунтуванням простоти супроводу); рівні згідно запиту клієнта: **адміністратор** (повний доступ до налаштувань чату та всіх модераторських дій), **модератор** (модераторські функції без повного адмін-конфігу), **VIP** (доступ до окремих кімнат + зняті/підвищені ліміти на дописи порівняно зі звичайним користувачем), **зареєстрований** (завантаження зображень у чат, налаштування власного профілю — узгодити з T10/T18), **гість** (обмежений набір); модель даних: міграції/поля або таблиці для ролі користувача та **ACL кімнат для VIP** (напр. членство/мінімальний tier кімнати); інтеграція з існуючою модерацією **T12** і авторизацією **T04/T05** (REST + `routes/channels.php`); публічні відповіді API для списків користувачів / `me` містять безпечні ознаки для UI (**колір плашки:** черновий адмін, зелений мод, оранжевий VIP — поле або мапінг на фронті за каноном з задачі); оновити **`docs/chat-v2/openapi.yaml`**; не розкривати зайвий PII гостям (узгоджено з **T22**)
- **QA evidence:** [docs/chat-v2/T21-QA.md](../docs/chat-v2/T21-QA.md) — `php artisan test` + `npm run build` PASS; сценарії VIP-кімнати, гість vs зображення, мод vs адмін у `/api/v1/mod/*`
